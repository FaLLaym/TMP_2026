### 3. Ответы на контрольные вопросы

#### 1. Для чего нужна команда `git init, git clone`?

- `git init`
  Эта команда создаёт в текущем каталоге новый скрытый подкаталог с именем `.git`, содержащий все необходимые файлы репозитория. Теперь каталог, в котором находится `.git` может стать репозиторием с контролем версий git.
- `git clone`
  Клонирует репозиторий из открытого источника.

#### 2. Что такое "коммит"?

Коммит - это фиксация всех проиндексированных(`git add`) изменений в проекте. Коммит выполняется при помощи команды `git commit`
Коммит состоит из: - Снимок **состояния** всех файлов - Автора изменений(имя и почта) - Дата и время коммита - Сообщение с описанием изменений

#### 3. Чем отличаются файлы `.puml` от `.png` в контексте вашей работы?

`.puml` это текстовый файл, по которому интерпретатор может нарисовать человекопонятную схему в формате изображения. Например `.png`, как в нашем случае.

`.png` уже готовое изображение, которое уже не получится так просто изменить, как текстовый `.punl`

Поскольку git это контроль версий текстовых файлов, то лучше хранить именно исходный `.puml` файл. Да и нет смысла каждый раз сохранять изображение схемы проекта, который в любой момент может изменится.

#### 4. Объясните, как в коде Go реализуется связь "класс User использует интерфейс Notifier", показанная на диаграмме.

Интерфейс:

```go
  //находится на строка 6-8
  type Notifier interface {
    Send(message string) error
}
```

Реализации - это структуры, которые используют метод `Send`

```go
// Send реализует интерфейс Notifier для EmailNotifier.
func (e EmailNotifier) Send(message string) error {
    fmt.Printf("Sending email via %s:%d: %s\n", e.smtpHost, e.smtpPort, message)
    // Здесь была бы реальная отправка email
    return nil
}

//...

// Send реализует интерфейс Notifier для SmsNotifier.
func (s SmsNotifier) Send(message string) error {
    fmt.Printf("Sending SMS to %s using API key %s: %s\n", s.phoneNumber, s.apiKey, message)
    // Здесь была бы реальная отправка SMS
    return nil
}
```

`User` получает нужную реализацию

```go
func main() {
    // Создаем два способа уведомления
    emailNoti := EmailNotifier{smtpHost: "smtp.gmail.com", smtpPort: 587}
    smsNoti := SmsNotifier{apiKey: "12345", phoneNumber: "+79991112233"}
```

Преимущество такого подхода в том, пользователь не создаёт уведомитель сам, а получает его извне. Это позволяет гибко подставлять разные реализации без изменения кода `User`. User не зависит конкретно от `EmailNotifier` или `SmsNotifier`. При таком подходе достаточно настроить сам интерфейса уведомлений `Notifier`
